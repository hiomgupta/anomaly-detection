## **🏆 Executive Summary**

### **Overview**

The **Intelligent Banking Document Fraud Detection System** is an **AI-powered, offline-capable** solution designed to **detect tampering, forgery, and anomalies** in banking documents (land records, bank statements, legal documents, and financial statements) **in real time**. The system leverages **multi-layered detection techniques**, including **image forensics, PDF forensics, OCR consistency checks, and deep learning**, to provide **explainable, accurate, and actionable fraud insights** for underwriting teams.

This solution directly addresses the **SuRaksha Hackathon’s** call for **behavior-based authentication and real-time fraud detection** in fintech, while ensuring **privacy, compliance, and seamless user experience**.

3. **Explainable AI** with heatmaps, confidence scores, and audit trails.
4. **Offline operation** (no external APIs, compliant with hackathon rules).
5. **Seamless integration** with Canara Bank’s underwriting workflows.

## **🎯 Problem Statement**

### **Current Challenges in Banking Document Verification**

1. **Manual Verification is Slow and Error-Prone**
  - Banks rely on **human reviewers** to verify land records, bank statements, and legal documents.
  - **Time-consuming**: Takes hours to days for high-volume underwriting.
  - **Prone to human error**: Fatigue, oversight, and bias lead to missed forgeries.
2. **Digital Forgeries Are Sophisticated**
  - Fraudsters use **Photoshop, PDF editors, and AI tools** to tamper with documents.
  - Common techniques:
    - **Copy-move forgery** (duplicating regions in a document).
    - **JPEG compression artifacts** (editing and re-saving images).
    - **PDF structural tampering** (hidden layers, font mismatches, metadata edits).
    - **OCR inconsistencies** (logical errors in extracted text).
3. **Existing Solutions Are Insufficient**
  - Most tools rely on **single-layer detection** (e.g., only OCR or only image hashing).
  - **No explainability**: Black-box AI models lack transparency for underwriters.
  - **External API dependencies**: Many solutions require cloud-based LLM/OCR APIs (violates hackathon rules).
4. **Regulatory and Compliance Risks**
  - Banks face **legal and financial penalties** for approving fraudulent documents.
  - **Lack of audit trails** makes it difficult to prove due diligence.

### **Why This Matters for Canara Bank**

- **Financial losses**: Fraudulent documents lead to **bad loans, chargebacks, and reputational damage**.
- **Operational inefficiency**: Manual verification slows down **loan approvals and onboarding**.
- **Customer trust**: Fraud erodes confidence in digital banking services.

## **💡 Proposed Solution**

### **System Overview**

The **Intelligent Banking Document Fraud Detection System** is a **multi-layered, offline-capable** platform that **automatically detects tampering and forgery** in banking documents using:

1. **Image Forensics Layer** (for visual tampering).
2. **PDF Forensics Layer** (for structural tampering).
3. **OCR Consistency Layer** (for logical inconsistencies).
4. **Deep Learning Layer** (for pixel-level tampering detection).
5. **Scoring & Explainability Layer** (for actionable insights).

The system is built using a **modular microservices architecture** (FastAPI + React + PostgreSQL + Docker) and **runs entirely offline**, complying with the hackathon’s **no-external-API rule**.

  - User uploads a document (JPEG, PNG, PDF) via the React dashboard.
  - File is sent to the FastAPI backend (`/upload` endpoint).
2. **Preprocessing**
  - Convert PDFs to images (if needed).
  - Normalize resolution, color space, and size.
3. **Parallel Detection Layers**
  - **Image Forensics Service**:
    - Runs ELA, Copy-Move, Edge, and Wavelet analysis.
    - Returns scores and heatmaps for each technique.
  - **PDF Forensics Service**:
    - Checks for hidden revisions, EOF markers, and font mismatches.
    - Returns a PDF integrity score.
  - **OCR Service**:
    - Extracts text using EasyOCR.
    - Validates cross-field consistency (e.g., dates, amounts).
    - Checks font consistency.
  - **Deep Learning Service**:
    - Runs pixel-level tampering detection using pretrained models.
    - Generates Grad-CAM heatmaps.

### **2. ELA (Error Level Analysis) (from DocAuth)**
```python
# backend/app/services/image_forensics/ela.py
import cv2
import numpy as np

def run_ela(image_path, quality=95):
    original = cv2.imread(image_path)
    temp_path = "temp_ela.jpg"
    cv2.imwrite(temp_path, original, [int(cv2.IMWRITE_JPEG_QUALITY), quality])
    resaved = cv2.imread(temp_path)
    ela = cv2.absdiff(original, resaved)
    ela = cv2.convertScaleAbs(ela)
    heatmap = cv2.applyColorMap(ela, cv2.COLORMAP_JET)
    _, threshold = cv2.threshold(cv2.cvtColor(ela, cv2.COLOR_BGR2GRAY), 30, 255, cv2.THRESH_BINARY)
    score = (np.sum(threshold > 0) / threshold.size) * 100
    return score, heatmap
```

# backend/app/services/pdf_forensics.py
def run_pdf_forensics(pdf_path):
    with open(pdf_path, 'rb') as f:
        content = f.read()
    eof_count = content.count(b'%%EOF')
    score = 100 if eof_count > 1 else 0
    flags = ["Multiple EOF markers detected"] if eof_count > 1 else []
    return score, flags

import easyocr

reader = easyocr.Reader(['en'])  # Initialize once (global)

def run_ocr_analysis(image_path):
    results = reader.readtext(image_path)
    extracted_text = " ".join([result[1] for result in results])
    flags = []

    # Basic consistency checks (add more as needed)
    if "2027" in extracted_text or "2028" in extracted_text:
        flags.append("Future date detected")

    # Simple score: 100 if no flags, else 0 (improve later)
    score = 0 if flags else 100
    return score, flags

### **5. Weighted Scoring (Custom)**
```python
# backend/app/services/scoring.py
def generate_final_score(ela_score, pdf_score, ocr_score):
    weights = {
        'ela': 0.5,    # Image forensics = 50%
        'pdf': 0.3,    # PDF forensics = 30%
        'ocr': 0.2     # OCR consistency = 20%
    }
    final_score = (
        ela_score * weights['ela'] +
        pdf_score * weights['pdf'] +
        ocr_score * weights['ocr']
    )
    return final_score
```
